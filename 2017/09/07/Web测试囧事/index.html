测试输入框的边界

二分输入法

要义就是对折对折,不断缩小范围, 第一次输入0->OK, 第二次输入100->Not Ok, 第三次输入50……

边界值法

找边界,在边界上做文章,如边界是5,则试着输入6(B+1),5(B),4(B-1)

测试实践

虎扑论坛-马刺分论坛

  • 在页面跳转输入框输入非法字符，显示页面不存在，并自动跳转到论坛首页
  • 输入101（此时页面显示只有100页），跳转到page101，但是该页面不再有跳转链接
  • 输入100，跳转到100页，该页面有页面跳转
  • 输入1000，无法输入，位数限制应为3位。
  • 使用二分法找到系统最后一页为621， 最后一贴的回帖时间为2005.6.08
  • 跳转到火箭分论坛，输入999，页面有内容，说明该专区帖子比马刺多，同时说明该论坛固定给每一个分论坛划分1-999页。按照最新回帖时间依次往后排（当然，里面还有一些加权的排序）

JavaScript 的加载

js的执行线程和浏览器的渲染线程是互斥的，并且JS默认第阻塞加载的。页面的下载和渲染都必须停下来等待脚本执行完成。js执行过程耗时越长，浏览器等待响应用户输入的时间就越久

  • 所以在LR的时候，有很多优化是针对JS的，系统认为JS数量过多，应该合并。
  • 解决这个问题的办法还有一个思路：将JS放在onload事件外，这就意味着在window对象的onload结束后再开始触发下载脚本
  • 由于JS的下载和执行的时间差让一些功能存在漏洞，比如某一个页面的跳转button，正常来讲某一个js禁用了他。但是由于网络原因，js一直没有加载出来，那么用户看到这个button就是可点的，于是就跟原来设计的程序不符了

每当一段JS脚本下载后，都会立刻执行该文件。因此两个JS请求并不是无缝连接的，而是隔了一段执行时间

重复验证button

我们经常需要验证一个button的功能，如跳转，如提交表格等，当点击一次以后，在页面刷新之前应该进行多次点击以确保该按钮没有重复提交功能
